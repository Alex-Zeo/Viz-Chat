# Viz-Chat Demo Script (3 minutes)

## Pre-demo Setup (before recording)
1. Start Chrome: `/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=/tmp/cr-chrome-profile --headless=new`
2. Start server: `DEMO_MODE=1 npx tsx --env-file .env server/index.ts`
3. Start frontend: `npx vite --host`
4. Open http://localhost:5173 in browser
5. Pre-warm: Run "Show me revenue trends for olist" once so the dashboard is populated
6. Clear the sidebar chat history (refresh if needed)

## Demo Flow

### 0:00-0:20 — Hook
- "What if your data dashboard could build itself? No drag-and-drop. No config files. Just ask a question and watch 4 AI agents race to answer it."
- Show the empty 4-panel grid

### 0:20-1:30 — Live Query
- Type in sidebar: **"What are the revenue trends for Olist?"**
- Narrate as pipeline stages light up:
  - "Parse detects revenue + trend intent"
  - "Probe scans the database schema"
  - "Rank picks 4 complementary chart types"
  - "Assign gives each agent its data slice"
  - "Now 4 agents build in parallel — each one iterating with a quality ratchet"
- Point out iteration tabs appearing on panels
- Point out PQI scores streaming in sidebar tool cards
- Wait for convergence + "Generated at runtime" badge

### 1:30-2:10 — Panel Refinement (the differentiator)
- "But what if you want to tweak one panel? You can talk to it directly."
- Type: **"/panel 2 make the bars horizontal and use a darker color scheme"**
- Watch panel 2 re-run its Ralph loop with the user's feedback
- "One agent re-runs, the others stay locked. Human-in-the-loop refinement."

### 2:10-2:40 — Architecture Callout
- "Under the hood: 7-stage orchestrator, AG-UI protocol for real-time streaming, CopilotKit for the chat interface, and a Karpathy ratchet — each panel iterates until quality converges."
- Point to status bar: agents, iterations, PQI, DQI, wall clock
- "All 4 panels are ECharts configs generated by Claude, rendered in headless Chrome, evaluated by DOM + vision checks"

### 2:40-3:00 — Close
- "Self-assembling dashboards. Ask a question, get a dashboard. Refine any panel by talking to it. That's Viz-Chat."

## Key Talking Points
- 4 parallel agents (not sequential)
- Karpathy ratchet = iterative quality improvement (like RLHF but for charts)
- Human-in-the-loop via /panel N
- AG-UI protocol streams real-time progress
- No templates — every chart is generated from scratch at runtime

## Backup Plan
- If the live query is slow: have the pre-warmed dashboard visible, skip to /panel refinement
- If Chrome fails: show the sidebar tool call stream, explain the architecture
- If /panel fails: "Panel refinement re-runs the same quality loop with your feedback injected"
